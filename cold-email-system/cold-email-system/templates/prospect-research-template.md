# 客户背景调研模板

## 发开发信前，先花5分钟调研目标客户

### 第一步：公司信息

| 项目 | 内容 | 来源 |
|------|------|------|
| 公司全名 | | Google |
| 所在州/城市 | | Google |
| 所属行业 | | LinkedIn |
| 公司规模 | | LinkedIn / ZoomInfo |
| 年营收估算 | | ZoomInfo / SimilarWeb |
| 主要产品 | | 官网 |

### 第二步：关键决策人

| 项目 | 内容 | 来源 |
|------|------|------|
| 采购经理姓名 | | LinkedIn |
| 职位 | | LinkedIn |
| 在职时间 | | LinkedIn |
| 邮箱（推测） | | Hunter.io / RocketReach |
| 工程负责人 | | LinkedIn |

### 第三步：竞品/现有供应商线索

| 项目 | 内容 | 来源 |
|------|------|------|
| 提到过的供应商 | | Google搜索 "公司名 + supplier/vendor" |
| 参加的展会 | | 展会官网参展商名单 |
| 社交媒体动态 | | LinkedIn / Facebook |

### 第四步：触发事件（Timely Trigger）

> 有触发事件的开发信回复率是普通邮件的 **3-5倍**

| 触发事件 | 如何发现 |
|---------|---------|
| 新产品发布 | 官网新闻 / LinkedIn |
| 融资/扩张 | Crunchbase / LinkedIn |
| 投诉现有供应商 | Google Reviews / Reddit / 贸易论坛 |
| 新员工入职（采购岗） | LinkedIn |
| 季节性采购周期 | 行业日历 |

### 第五步：个性化钩子

根据以上调研，填入开发信的个性化部分：

```
[Company] 最近 [触发事件]，这意味着他们可能需要 [推测需求]。
联系 [Key Person]，因为 [原因]。
邮件中提到 [具体细节] 来展示我做了功课。
```
