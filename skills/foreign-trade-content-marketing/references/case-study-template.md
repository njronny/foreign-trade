# 客户案例模板

## 结构: Challenge → Solution → Results

## 标题公式
- "How We Helped [Client Type] [Achieve Result] with [Product/Service]"
- "[Result]: A [Industry] Case Study"
- "[Problem]? Here's How We Solved It for [Client Type]"

## 模板

### 开头钩子（2-3句）
用一个惊人的数据或对比开篇：
"When [Company Type] approached us, they were [problem]. [X months] later, [result]."

### The Client（匿名背景）
```
**Industry**: [e.g., Hydraulic Equipment Manufacturing]
**Location**: [e.g., Midwest, USA]
**Company Size**: [e.g., 50-100 employees]
**Product**: [e.g., Directional control valves]
```
> 不透露真实公司名，用行业+地区描述

### The Challenge
```
[描述客户遇到的具体问题，2-3段]

### Requirements:
- [需求1: 具体技术参数]
- [需求2: 交期/成本要求]
- [需求3: 质量标准]
- [需求4: 特殊要求]

### Previous Supplier Issues:
- [之前供应商的问题1]
- [之前供应商的问题2]
```

### Our Solution
```
[描述我们的解决方案，重点展示专业性]

### Process:
1. [步骤1: 如收到图纸后的DFM分析]
2. [步骤2: 如材料选择建议]
3. [步骤3: 如首件打样+检测]
4. [步骤4: 如批量生产+质量管控]

### Technical Details:
| Parameter | Specification |
|-----------|--------------|
| Material | [具体材料] |
| Tolerance | [公差要求] |
| Surface Finish | [表面处理] |
| Quantity | [数量] |
| Lead Time | [实际交期] |

### Quality Control:
- [检测方法1]
- [检测方法2]
- 检测报告随货发送
```

### The Results
```
[用量化数据展示成果]

| Metric | Before | After | Improvement |
|--------|--------|-------|-------------|
| Per-part cost | $X | $Y | -Z% |
| Lead time | X weeks | Y days | -Z% |
| Quality rate | X% | Y% | +Z% |

### Client's Words:
> "[客户原话，包含具体感受和量化描述]"
> — [Title], [Industry] Company, [Region]
```

### Lessons Learned
```
[1-2个从这个项目中学到的经验，展示专业深度]
```

### CTA
```
Facing similar challenges? We can help.
📧 Send your drawing → Quote in 24 hours
🌐 [website]
```

## 案例写作规则
1. **必须包含量化数据** — 没有数据的案例不如不写
2. **用对比表格** — Before/After一目了然
3. **包含客户原话** — 哪怕是改写的，也要有"人话"
4. **不透露客户名称** — 用行业+地区代替
5. **突出技术细节** — 证明你不是空谈
6. **每篇聚焦一个核心成果** — 不要面面俱到
